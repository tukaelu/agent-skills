## 1. Frontmatter (YAML)

### name
- [ ] `[W]` 64文字以内である
- [ ] `[C]` 小文字、数字、ハイフンのみ使用している
- [ ] `[W]` 曖昧な名前を避けている（`helper`, `utils`, `tools` はNG）
- [ ] `[W]` 予約語を含まない（`anthropic`, `claude` はNG）

### description
- [ ] `[C]` 空でない
- [ ] `[W]` 1024文字以内である
- [ ] `[C]` スキルが**何をするか**を説明している
- [ ] `[C]` スキルを**いつ使うべきか**を説明している
- [ ] `[W]` 具体的なトリガーワードやシナリオを含んでいる
- [ ] `[I]` 具体的なユースケースを明示している

### オプションフィールドの適切な使用

| フィールド | 用途 |
|---|---|
| `allowed-tools` | スキルが使用するツールを明示的に許可（例: `Bash`, `Read`, `Write`） |
| `disable-model-invocation: true` | 副作用のあるワークフロー（deploy, commit 等）でモデルによる自動起動を防ぐ |
| `user-invocable: false` | ユーザーが `/` メニューから呼ぶべきでない背景知識スキルに設定 |
| `context: fork` | 独立した処理をサブエージェントで実行する場合 |
| `agent` | `context: fork` 時に使用するエージェント型（`Explore`, `Plan`, `general-purpose` など） |
| `argument-hint` | 引数を受け取るスキルに補完ヒントを表示（例: `[issue-number]`） |
| `model` | スキル実行時に使用するモデルを指定 |
| `hooks` | スキルのライフサイクルにフックを設定 |

- [ ] `[C]` スキルが特定のツールを必要とする場合、`allowed-tools` にそのツールを明示的に列挙している
- [ ] `[W]` 副作用のあるワークフロー（deploy, commit, send 系）に `disable-model-invocation: true` を設定している
- [ ] `[I]` 引数を受け取るスキルに `argument-hint` を設定している
- [ ] `[I]` 独立した処理には `context: fork` を検討している

### 変数と動的コンテキスト

| 変数 | 説明 |
|---|---|
| `$ARGUMENTS` | スキル呼び出し時に渡された全引数 |
| `$ARGUMENTS[N]` / `$N` | 位置指定で引数を取得（0始まり） |
| `${CLAUDE_SESSION_ID}` | 現在のセッションID |
| `` !`command` `` | シェルコマンドを事前実行しその出力を埋め込む（動的コンテキスト注入） |

- [ ] `[W]` 引数を扱う場合 `$ARGUMENTS` 系変数を使用している（ハードコード禁止）
- [ ] `[I]` 動的なコンテキストが必要な場合 `` !`command` `` 構文を検討している

## 2. SKILL.md Body

### 簡潔さ (Conciseness)
- [ ] `[W]` 500行以下である
- [ ] `[W]` Claude がすでに知っている情報を含まない
- [ ] `[I]` 冗長な説明より簡潔な例を優先している
- [ ] `[I]` 各情報がトークンコストに見合う価値がある

### 構造
- [ ] `[W]` 明確なセクション分けがある
- [ ] `[I]` 目次またはナビゲーションがある（長い場合）
- [ ] `[W]` 「When to Use This Skill」セクションが body にない（description に含めるべき）

### Progressive Disclosure
- [ ] `[W]` 詳細な情報は別ファイルに分離している
- [ ] `[W]` 参照ファイルの用途が明確に説明されている
- [ ] `[I]` 参照ファイルへのリンクが1階層深さまで
- [ ] `[I]` 100行以上の参照ファイルには目次がある

## 3. Workflows

### ワークフロー設計
- [ ] `[W]` 複雑な操作は明確な順序のステップに分解している
- [ ] `[W]` 条件分岐が明確に記述されている
- [ ] `[I]` 決定ポイントで明示的なガイダンスがある

### バリデーション
- [ ] `[W]` 重要な操作後に検証ステップがある
- [ ] `[W]` エラーメッセージが具体的で問題箇所を特定できる
- [ ] `[I]` 「実行→検証→修正→繰り返し」のフィードバックループがある

---

## 4. Scripts

スクリプトを含むスキルのみ評価する。

### スクリプト品質
- [ ] `[C]` エラー条件を明示的に処理している
- [ ] `[W]` 定数に説明コメントがある（マジックナンバー禁止）
- [ ] `[W]` 依存パッケージが SKILL.md に記載されている
- [ ] `[W]` プラットフォーム制限が明記されている

### ファイルパス
- [ ] `[W]` フォワードスラッシュを使用している（`scripts/helper.py`）
- [ ] `[W]` Windows スタイルのパスを使用していない（`scripts\helper.py` はNG）
- [ ] `[I]` ファイル名が内容を示す説明的な名前である

---

## 5. References

参照ファイルを含むスキルのみ評価する。

- [ ] `[W]` 必要な時だけ読み込まれるよう設計されている
- [ ] `[W]` SKILL.md と情報が重複していない
- [ ] `[I]` ドメインまたは機能ごとに整理されている

---

## 6. Content Quality

### 用語と言語
- [ ] `[W]` 一貫した用語を使用している（同義語の混在を避ける）
- [ ] `[W]` 時間に敏感な情報を含まない
- [ ] `[I]` 命令形/不定詞形で記述している

### 例とテンプレート
- [ ] `[I]` 具体的な入出力例がある
- [ ] `[I]` 例のスタイルが一貫している

---

## 7. Anti-Patterns（避けるべきこと）

他セクションと重複しない固有のアンチパターン：

- [ ] `[W]` デフォルトなしで多くの選択肢を提示していない
- [ ] `[W]` 曖昧な説明やスキル名を使用していない
- [ ] `[I]` 深くネストしたファイル参照がない

---

## 8. MCP Tools

MCP ツールを使用するスキルのみ評価する。

- [ ] `[W]` 完全修飾ツール名を使用している（`ServerName:tool_name`）
- [ ] `[W]` サーバープレフィックスを省略していない

---

## Severity Levels

### Critical（必須修正）
スキルの正常な機能を妨げる問題。

### Warning（推奨修正）
スキルの効果を低下させる問題。

### Info（改善提案）
対応しなくても動作するが、品質向上につながる改善。
